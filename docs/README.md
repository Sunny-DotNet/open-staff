# OpenStaff 项目文档

本目录包含OpenStaff多智能体软件开发平台的项目文档。

## 📄 文档列表

### **[agent-runtime-lifecycle.md](./agent-runtime-lifecycle.md)** - 智能体创建 / 执行链路

面向调试的技术链路文档，包含：

- 🧭 **三条主线总览**: 角色保存、测试对话、正式会话
- 🔀 **共同汇合点**: `IAgentService`、`ApplicationAgentRunFactory`、`AgentFactory`
- 🧠 **逻辑导图**: Mermaid 总览图，快速定位入口与回流路径
- 🧪 **调试断点地图**: 常见问题对应的关键方法、对象和值
- 📌 **概念区分**: `AgentRole`、effective role、`ProjectAgent`、`MessageHandler`、`ChatFrame`

### **[PRD.md](./PRD.md)** - 产品需求文档 (Product Requirements Document)

完整的产品需求规格说明，包含：

- 📋 **项目概述**: 愿景、目标、用户画像
- 🚀 **功能需求**: 智能体协作、项目管理、代码生成、质量保证
- ⚙️ **非功能需求**: 性能、安全、可扩展性、兼容性
- 📊 **数据模型**: 实体关系图、数据字典
- 🔌 **API设计**: RESTful API规范、SignalR实时通信
- 🎨 **界面设计**: 布局原则、关键界面原型
- 🔐 **安全合规**: 数据安全、代码安全、业务合规
- 📈 **性能监控**: 系统指标、日志诊断
- 🚢 **部署运维**: 部署方式、容器化、配置管理
- 🎯 **成功指标**: KPI定义、评估方法
- 🗺️ **产品路线**: 分阶段发展计划
- 💡 **创新亮点**: 核心竞争优势

## 📖 使用指南

### **产品经理/项目经理**
1. 阅读完整PRD了解产品愿景
2. 参考用户画像定义目标用户群体
3. 根据功能需求规划产品迭代
4. 使用成功指标制定项目目标

### **开发团队**
1. 理解智能体协作系统架构
2. 参考API设计规范进行接口开发
3. 根据数据模型设计数据库表结构
4. 遵循界面设计规范实现前端功能

### **测试团队**
1. 根据功能需求编写测试用例
2. 参考非功能需求制定性能测试计划
3. 使用成功指标验证产品质量

### **UI/UX设计师**
1. 参考界面设计规范进行视觉设计
2. 根据交互原型设计用户体验流程
3. 结合智能体协作特点设计协作界面
